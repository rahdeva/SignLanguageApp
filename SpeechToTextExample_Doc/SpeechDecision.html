Latar Belakang

Mengapa dok

umen ini dibuat

SignLanguage adalah aplikasi aksesibilitas yang mentranskrip ucapan caregiver> secara real-time menjadi teks besar di layar, sehingga penyandang disabilitas nonverbal atau dengan keterbatasan motorik bisa mengikuti percakapan. Caregiver dan pengguna duduk berhadapan, perangkat diletakkan di antara mereka.

Kebutuhan inti: offline-first (audio tak boleh keluar perangkat demi privasi), partial/volatile results (kata-per-kata, bukan nunggu kalimat selesai), latensi rendah (ideal < 1 dtk), serta dukungan bahasa Indonesia (id-ID). Dokumen ini membandingkan dua pendekatan yang sepenuhnya berada dalam import Speech milik Apple, tanpa melirik ke Whisper atau cloud STT lain.

Ringkasan

SFSpeechRecognizer adalah jawabannya — untuk sekarang

Per Juni 2026, bahasa Indonesia (id-ID) tidak ada di SpeechTranscriber.supportedLocales (iOS 26+). Satu-satunya opsi di dalam framework Speech yang mendukung id-ID, on-device, partial results, dan custom vocabulary adalah SFSpeechRecognizer + SFSpeechAudioBufferRecognitionRequest dengan requiresOnDeviceRecognition = true.

SpeechAnalyzer/SpeechTranscriber adalah masa depan — arsitektur yang lebih modern, lebih mudah, dan hasilnya lebih stabil — tapi belum mencakup id-ID. Oleh karena itu kami mengadopsi pendekatan hybrid dengan versioning: engine default sekarang adalah SFSpeechCaptioner, dan ketika Apple menambah id-ID ke SpeechTranscriber (di versi iOS mendatang), SpeechEngineFactory akan otomatis beralih tanpa perubahan kode UI.

Analisis

M

engapa SFSpeechRecognizer menang — dengan satu syarat

4

.1 Dukungan id-ID

SFSpeechRecognizer.supportedLocales() mencakup id_ID sejak iOS 13. On-device recognition untuk id-ID tersedia di perangkat dengan Apple Neural Engine (A12+ / iPhone XR/XS ke atas). Sementara itu SpeechTranscriber.supportedLocales belum menyertakan id-ID. Ini adalah faktor penentu tunggal yang membuat SFSpeechRecognizer menjadi pilihan wajib untuk deployment sekarang.

Implikasi Fallback

Karena SpeechTranscriber tidak mendukung id-ID, tidak ada "fallback" di dalam SpeechAnalyzer — ia tidak bisa memproses bahasa yang modelnya tidak diinstal. Satu-satunya fallback yang valid adalah menggunakan SFSpeechRecognizer. Strategi hybrid di bab 6 memastikan transisi mulus ketika Apple menambah id-ID ke SpeechTranscriber di masa depan.

4.2

Partial / Volatile Results & Latensi

Kedua engine mendukung hasil sementara (kata-per-kata). SFSpeechRecognizer mengirimkan SFSpeechRecognitionResult dengan isFinal = false untuk partial; SpeechTranscriber mengirimkan segmen dengan reportingOptions: [.volatileResults]. SpeechAnalyzer unggul tipis dalam latensi karena pipeline-nya yang lebih modern dan dedicated, tapi selisihnya tidak signifikan dalam konteks live caption (keduanya di bawah 1 dtk).

4

.3 Custom Vocabulary — Poin Kritis untuk Domain Medis

Ini adalah kelemahan terbesar SpeechTranscriber untuk kasus penggunaan kami. SFSpeechRecognizer mendukung contextualStrings> — array hingga ~1000 frasa yang memberi bobot lebih tinggi pada istilah tertentu saat decoding. Untuk aplikasi aksesibilitas yang menangani nama obat (Amlodipine, Salbutamol, Metformin), nama pasien, dan istilah medis lain yang rawan salah transkrip, fitur ini esensial.

SpeechTranscriber tidak memiliki API contextual strings versi apapun. Satu-satunya mekanisme pengaruh adalah model bahasa bawaan dan data pengguna dari sistem, yang tidak bisa dikontrol oleh developer.

Catatan: contextualStrings membantu, tapi tidak menjamin akurasi 100%. Tetap perlu validasi dan pelatihan pengguna bahwa caption yang muncul mungkin perlu dikoreksi.

4

.4 Offline / On-device

SFSpeechRecognizer: tersedia on-device sejak iOS 17 untuk beberapa locale. id-ID mendukung on-device recognition di perangkat A12+ — tetapi wajib diuji di perangkat target karena model bahasa perlu diunduh sekali (≈ 50–120 MB). Gunakan recognizer.supportsOnDeviceRecognition untuk deteksi.

SpeechTranscriber: dirancang offline sejak awal dengan AssetInventory untuk manajemen model. Preset .offlineTranscription memastikan audio tidak pernah meninggalkan perangkat.

4

.5 Kompleksitas & Perawatan

SFSpeechRecognizer memerlukan rotasi segmen manual — segmen recognition task memiliki batas durasi ~1 menit. Untuk captioning kontinu, kita perlu mendeteksi jeda bicara (silence timer) atau batas waktu, lalu memulai segmen baru. Ini menambah kompleksitas tetapi sudah teruji di banyak aplikasi produksi.

SpeechTranscriber menangani segmen secara otomatis — lebih mudah dirawat. Namun, ia memerlukan BufferConverter untuk menyesuaikan format audio mic ke format yang diminta analyzer, dan AssetInventory untuk download model.

.2 Engine Utama: SFSpeechCaptioner

Engine

default untuk id-ID saat ini. Menangani AVAudioEngine → recognition → partial → commit + rotasi segmen.

Swift SignLanguageApp/SFSpeechCaptioner.swift 
@MainActor
final class SFSpeechCaptioner: LiveCaptioner {

    static let engineName = "SFSpeechRecognizer"

    // Callbacks
    var onPartial: ((String) -> Void)?
    var onCommit: ((String) -> Void)?
    var onLevel: ((Float) -> Void)?
    var onStatus: ((CaptionEngineStatus) -> Void)?

    private let recognizer: SFSpeechRecognizer?
    private let contextualStrings: [String]
    private let audioEngine = AVAudioEngine()
    private unowned let sink = AudioSink()
    private var request: SFSpeechAudioBufferRecognitionRequest?
    private var task: SFSpeechRecognitionTask?

    // Rotasi segmen
    private var latestPartial = ""
    private let silenceCommitDelay: TimeInterval = 1.4
    private let maxSegmentDuration: TimeInterval = 50

    func start() async throws {
        // 1. Pastikan izin (prepare sudah dipanggil sebelumnya)
        guard let recognizer, recognizer.isAvailable else {
            throw CaptionError.recognizerUnavailable
        }
        // 2. Audio session — voice processing untuk echo cancel & noise suppression
        try AVAudioSession.sharedInstance().setCategory(
            .playAndRecord, mode: .voiceChat,
            options: [.duckOthers, .allowBluetoothHFP, .defaultToSpeaker]
        )
        // 3. Tap audio → recognition request
        try startAudioEngineIfNeeded()
        beginSegment(with: recognizer)  // partial + final handler
        onStatus?(.listening)
    }

    private func beginSegment(with recognizer: SFSpeechRecognizer) {
        let req = SFSpeechAudioBufferRecognitionRequest()
        req.shouldReportPartialResults = true
        req.taskHint = .dictation
        if recognizer.supportsOnDeviceRecognition {
            req.requiresOnDeviceRecognition = true  // ← privasi
        }
        req.contextualStrings = contextualStrings  // ← custom vocab
        task = recognizer.recognitionTask(with: req) { [weak self] result, error in
            Task { @MainActor in self?.handle(result: result, error: error) }
        }
    }

    // Partial → UI, Final → commit + rotasi segmen baru
    private func handle(result: SFSpeechRecognitionResult?, error: Error?) {
        if let result {
            if result.isFinal {
                commit(result.bestTranscription.formattedString)
                rotateSegment()  // mulai segmen baru
            } else {
                latestPartial = result.bestTranscription.formattedString
                onPartial?(latestPartial)  // ← update UI real-time
                scheduleSilenceCommit()  // jeda 1,4 dtk → commit
            }
        }
    }
}

enum SpeechEngineFactory {

    /// Cek dukungan locale di SpeechTranscriber (iOS 26+).
    static func analyzerSupports(locale: Locale) async -> Bool {
        guard #available(iOS 26.0, *) else { return false }
        let supported = await SpeechTranscriber.supportedLocales
        let target = locale.identifier(.bcp47)
        return supported.contains { $0.identifier(.bcp47) == target }
    }

    /// Pilih engine terbaik untuk locale.
    /// - SpeechAnalyzer jika iOS 26+ DAN locale ada di supportedLocales.
    /// - Selain itu SFSpeechRecognizer (kasus id-ID saat ini).
    @MainActor
    static func make(
        localeIdentifier: String = "id-ID",
        contextualStrings: [String] = DomainVocabulary.medical
    ) async -> LiveCaptioner {
        let locale = Locale(identifier: localeIdentifier)
        if #available(iOS 26.0, *),
           await analyzerSupports(locale: locale) {
            return SpeechAnalyzerCaptioner(locale: locale)
        }
        return SFSpeechCaptioner(locale: locale, contextualStrings: contextualStrings)
    }
}

/// Live stream: baris final + partial + caret berkedip.
struct CaptionStream: View {
    @Bindable var vm: CaptionViewModel
    let bottomID = "bottom"

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 8) {
                ForEach(vm.transcript) { line in
                    Text(line.text)
                        .font(.system(size: vm.captionPointSize, weight: .bold))
                        .foregroundStyle(vm.palette.ink)
                        .frame(maxWidth: .infinity, alignment: .leading)
                }

                if !vm.partial.isEmpty {
                    PartialLine(text: vm.partial, palette: vm.palette,
                               caretOn: vm.isListening)
                }

                Color.clear.frame(height: 1).id(bottomID)
            }
            .padding(.horizontal, Metrics.screenMargin)
            .padding(.vertical, 16)
        }
        .scrollIndicators(.hidden)
        .onChange(of: vm.partial) { _, _ in
            scrollToBottom()
        }
    }
}

/// Partial text + caret berkedip — terlihat seperti sedang diketik.
struct PartialLine: View {
    let text: String
    let palette: Palette
    let caretOn: Bool

    var body: some View {
        HStack(spacing: 0) {
            Text(text)
                .font(.system(size: 44, weight: .bold))
                .foregroundStyle(palette.partial)
                .italic()
            if caretOn {
                Caret(color: palette.accent, on: caretOn)
            }
        }
        .frame(maxWidth: .infinity, alignment: .leading)
        .transition(.opacity)
    }
}

struct Caret: View {
    let color: Color
    let on: Bool
    @State private var blink = false

    var body: some View {
        Rectangle()
            .fill(color)
            .frame(width: 3, height: 48)
            .opacity(blink ? 1 : 0.15)
            .animation(.easeInOut(duration: 0.6).repeatForever(autoreverses: true),
                       value: blink)
            .onAppear { blink = on }
    }

Edge Cases

    
Hal
-hal yang perlu diantisipasi

    
7
.1 Ruangan berisik / jarak mic-ke-mulut
    

      Caregiver bisa duduk 0,5–2 meter dari perangkat. Gunakan AVAudioSession.Mode.voiceChat
      yang mengaktifkan echo cancellation, noise suppression, dan
      AGC (Automatic Gain Control) bawaan sistem. Aktifkan juga
      inputNode.setVoiceProcessingEnabled(true) untuk peredaman bising perangkat keras.
      Pantau level audio via AudioSink.rms() dan tampilkan banner "Suara belum terdengar jelas"
      bila level di bawah threshold selama > 3 detik (ini sudah diimplementasikan di SFSpeechCaptioner).
    


    
7
.2 recognizer.isAvailable = false
    

      SFSpeechRecognizer.isAvailable bisa false jika model bahasa belum diunduh,
      server tidak terjangkau (mode server-based), atau resource sistem sedang penuh. Strategi:
    

    
    
      
  • Cek isAvailable sebelum start() dan tampilkan error spesifik.
    • 
      
  • Untuk on
    • -device: pastikan model sudah diunduh via SFSpeechRecognizer.requestAuthorization
        (iOS menangani unduhan otomatis). Beri UI progress jika perlu.
      
  • Sediakan tombol Coba lagi yang memanggil prepare() + start() ulang.
    • 
    


    
7
.3 Izin ditolak
    

      Dua izin diperlukan: Speech Recognition (SFSpeechRecognizer.requestAuthorization)
      dan Microphone (AVAudioApplication.requestRecordPermission).
      Jika salah satu ditolak:
    

    
    
      
  • Pertama kali: tampilkan PermissionView dengan penjelasan ramah dan tombol izinkan.
      
  • Jika
    •  sudah pernah ditolak (status .denied): arahkan ke Settings.app dengan
        UIApplication.openSettingsURLString. Jangan loop meminta izin — itu tidak akan muncul lagi.
      
  • P
    • ertimbangkan partial experience: jika mic diizinkan tapi speech tidak, setidaknya
        tampilkan waveform audio sebagai indikator bahwa perangkat "mendengar".
    


    
7
.4 Rotasi segmen (batas ± 1 menit)
    

      SFSpeechRecognitionTask memiliki batas durasi sekitar 1 menit per segmen.
      Strategi rotasi:
    

    
    
      
  • Silence timer: 1,4 detik tanpa deteksi suara → commit segmen & mulai baru.
        Ini juga membuat caption terasa responsif: kalimat pendek langsung muncul.
    • 
      
  • Max duration: 50 detik — rotasi paksa sebelum batas 1 menit.
        Partial terakhir di-commit sebagai final, segmen baru dimulai tanpa menghentikan audio engine.
    • 
      
  • G
    • unakan DispatchWorkItem yang di-cancel bila ada aktivitas suara baru
        (reset silence timer setiap partial result diterima).
    


    
7.5 Multiple caregiver / bergantian bicara
    

      SFSpeechRecognizer tidak mendukung diarisasi / speaker identification di dalam satu segmen.
      Jika dua orang bergantian bicara, hasilnya akan menjadi satu aliran teks tanpa label pembicara.
      Untuk versi awal ini cukup — aplikasi fokus ke satu caregiver utama.
      Di masa depan, SpeechTranscriber mendukung reportingOptions: [.speakerIdentification]
      (iOS 26+) untuk membedakan pembicara.
    


    
7.6 Aksesibilitas UI
    

      Caption harus bisa dibaca dari jarak 1–2 meter oleh pengguna dengan keterbatasan motorik.
      Desain aplikasi sudah mengatur baseCaptionSize = 44pt dengan rentang skala 0,75× – 1,5×
      (33 – 66pt). Teks partial menggunakan italic dengan warna redup (palette.partial) sementara
      teks final menggunakan weight bold dengan kontras tinggi (palette.ink).
      Pastikan Dynamic Type tidak dinonaktifkan agar pengaturan aksesibilitas sistem tetap berlaku.
